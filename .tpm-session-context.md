## Remaining Work (priority order)

### 1. tpm Profile Defaults (~1h)

Add a way to create a profile pre-configured for TPM use:

- Worktrees enabled by default
- Group naming: `tpm-{task}`
- Worktree naming: `tpm-{task}-{id}`

Look at `src/session/profile_config.rs` and `src/cli/profile.rs` to see how
profiles work. Either add a `--template tpm` flag to `aoe profile create` or
ship a `docs/profiles/tpm-recommended.toml` users copy into their profile dir.

### 2. TUI Checkbox for TPM Mode at Session Creation (~3-4h)

When creating a new session in the TUI, present a "Use TPM workflow" checkbox.
If checked:

- Verify `contrib/tpm-workflow/agents/orchestrator.md` (or installed plugin
  path) is accessible
- Modify the spawned `claude` command to add
  `--system-prompt "$(cat .../agents/orchestrator.md)"`
- The user's task description becomes the initial `--message`

Files to modify (probably):
- `src/tui/dashboard/add_session/` for the dialog UI
- `src/cli/add.rs` for the underlying logic
- May need a new struct field `tpm_mode: bool` on `AddArgs` and the TUI state

### 3. Run the Daemon Automatically in TUI Mode (~30 min)

When a user has TPM-mode sessions in the active profile, the sweeper should
run automatically. Check `src/main.rs` for the TUI entry path:

```rust
None => tui::run(&profile, debug_log_warning).await,
```

Spawn a tokio task that calls `events::run_sweeper(profile, Duration::from_secs(2))`
when the TUI starts. Make sure it's cancelled cleanly on TUI exit.

### 4. End-to-end test of the full workflow (~1h)

Once the above is in place:

1. Build: `cargo build --release`
2. Install plugin in Claude Code: `/plugin marketplace add Loulen/tpm-workflow`
3. Create a session via TUI with TPM mode on
4. Watch it dispatch a planner, accept the plan, dispatch implementers in parallel
5. Verify events flow correctly via `aoe events watch`
6. Document what worked and what broke

## Project Conventions

- Read `AGENTS.md` (symlinked as `CLAUDE.md`) for repo guidelines
- Use conventional commits: `feat:`, `fix:`, `docs:`, `refactor:`
- Run `cargo fmt` and `cargo clippy -- -D warnings` before committing — there
  is a husky pre-commit hook that enforces both
- Tests: `cargo test --lib events` for the events module
- Use `serial_test::serial` for tests that touch shared global state
