# Autonomous Continuation Prompt

You are running in a long-lived AoE-managed session with `--permission-mode bypassPermissions`. You have authorization to use any tool freely without asking for confirmation. Your job is to **continue implementing the remaining work in this repo until everything is done or you hit a true blocker**.

## Critical Operating Rules

1. **Do not stop after one item.** Implement, commit, push, then immediately move to the next item without summarizing or waiting for user input. The user is asleep — there is no one to respond. You must self-direct.
2. **Do not ask for confirmation.** You have full authorization. Just do.
3. **Commit and push after each completed item** so the user wakes up to incremental progress visible in git history.
4. **Run `cargo fmt`, `cargo clippy -- -D warnings`, and the relevant tests before each commit.** The repo has a husky pre-commit hook that enforces both — never use `--no-verify`.
5. **If `git push` is rejected (non-fast-forward)**, run `git pull --rebase origin tpm-mode` then re-push.
6. **Stop only if** (a) all items below are done and verified, OR (b) you hit a true blocker that requires a human decision (e.g., the husky hook keeps failing for reasons you can't fix, you'd need to delete user data, the repo is in an unrecoverable state).
7. **When you stop**, write a final commit `docs: autonomous run finished — see HANDOFF.md` with a `HANDOFF.md` file at the repo root summarizing what was done, what's left, and any decisions the user needs to make.

## Iteration Pattern

For each item:

1. Read the relevant existing code (use Read, Grep, Glob)
2. Make the change
3. `cargo build` — must succeed
4. `cargo test --lib <relevant>` — must pass
5. `cargo clippy -- -D warnings` — must pass
6. `cargo fmt`
7. `git add <specific files>` (no `git add .`)
8. `git commit -m "..."` (let husky run)
9. `git push origin tpm-mode`
10. Move to next item — DO NOT WAIT

If any step fails:
- Diagnose the failure
- Fix it
- Retry from the failed step
- After 5 failed attempts on the same step, write a HANDOFF.md noting the blocker and stop

## Blockers

These are real blockers (write HANDOFF.md and stop):
- Repo state is corrupted (uncommitted changes you didn't make, broken refs)
- Husky pre-commit fails for a reason you've tried 3+ ways to fix
- A test failure that requires architectural changes outside scope of remaining items
- The `tpm-mode` branch is gone or has diverged in ways you can't reconcile

These are NOT blockers (just keep going):
- A clippy warning — fix it
- A test failure — fix it
- Need to read more code to understand something — read it
- Need to run a command — run it
